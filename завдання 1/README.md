# Лабораторна робота №3

## Тема

Класичний книжковий шифр з двійковими координатами

## Мета роботи

- Реалізація класичного книжкового шифру з використанням двійкового представлення координат
- Вивчення принципів роботи шифру на основі ключового тексту
- Засвоєння механізмів перетворення символів у координати та зворотного відновлення

## Короткі теоретичні відомості

Книжковий шифр (Book Cipher) — це метод шифрування, при якому кожен символ відкритого тексту замінюється координатами його позиції в ключовому тексті. Кожна координата складається з номера рядка та номера позиції символу в рядку.

У даній реалізації координати представлені у двійковому вигляді: перші 8 біт відповідають номеру рядка (1-based), наступні 8 біт — номеру позиції в рядку (1-based). Загальна довжина адреси становить 16 біт.

## Опис алгоритму

### Шифрування

1. Ключовий текст розбивається на рядки за символом нового рядка
2. Для кожного символу відкритого тексту:
   - Виконується пошук першого входження символу в ключовому тексті
   - Визначаються координати: номер рядка (від 1) та позиція в рядку (від 1)
   - Координати перетворюються у двійковий формат: кожна координата кодується 8 бітами
   - Двійкові представлення координат об'єднуються в 16-бітну адресу
3. Результат — масив двійкових адрес

### Дешифрування

1. Для кожної двійкової адреси з масиву:
   - Адреса розбивається на дві частини по 8 біт
   - Перші 8 біт конвертуються в номер рядка, наступні 8 біт — в позицію
   - За координатами витягується символ з ключового тексту
2. Витягнуті символи об'єднуються в оригінальне повідомлення

## Структура проєкту

- `index.html` — головна сторінка з інтерфейсом користувача
- `script.js` — логіка шифрування та дешифрування книжковим шифром
- `style.css` — стилізація інтерфейсу

## Як запустити

1. Відкрийте файл `index.html` у браузері (подвійний клік або через контекстне меню "Відкрити за допомогою")
2. Вставте ключовий текст у поле "Ключовий текст"
3. Введіть JSON-повідомлення у відповідне поле
4. Натисніть кнопку "Encrypt" для шифрування або "Decrypt" для дешифрування

## Примітки / Обмеження

- Нумерація рядків та позицій починається з 1
- Шукається перше входження кожного символу в ключі
- Якщо символ не знайдено в ключовому тексті, виводиться помилка
- Пробіли та інші символи також шифруються
- Кожна координата обмежена 8 бітами (максимум 255 рядків та 255 позицій)
